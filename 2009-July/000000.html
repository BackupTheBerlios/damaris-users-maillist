> I am working on a system that uses a Spincore card for pattern
> generation. No development had been done on the systems for a long
> time. The older 2.4 kernel installation writes directly to a
> /dev/pulseblaster device driver. I can't find the source, but I
> believe that this driver came from your project. Has the driver been
> ported to the 2.6 kernel? I downloaded the damaris-0.11 source and see
> reference to a pulseblaster.ko driver, but have don't see the actual
> code for it.
Hi Russell!

Yes, it is(was) available. I've ported it last year, including automatic
loading of the driver.
make -C backends/drivers/SpinCore-PulseBlaster/  pulseblaster.ko

should compile the driver. The driver function did not change except for
one detail: The write function will give processor time to other
processes after each 16kb writing to prevent temporary system freezes
for writing long programs.

Udev is now used to setup permissions and links. The udev rules file can
be found at:
backends/tools/080_damaris.rules

**** unfortunately it does not compile with debian lenny (linux-2.6.26) ****
class_device_create and class_device_destroy are no longer available. I
will work on that at the weekend.

Achim
