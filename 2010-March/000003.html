Felipe Coelho wrote:
> Hi, is there a way to run gdb on the backend executable?
>
> I'm developing a backend and all the information I receive when running it
> through the Damaris frontend is that the "backend was terminated by signal
>  SIGSEGV".
>
> I've tried running it 'stand alone' like so:
>
> ~/Damaris/bin/ $ ../MyCore --spool /home/<myuser>/Damaris/bin/spool/
>
> I got the command line switch from a running backend which prints its argv
> at the beginning. When I run it, however, it just 'hangs', stat'ing a job
> file eternally and never getting the result it wants.
>   
Hi Felipe!

In deed it is intended to run the backend standalone.
Are the files job.000000000 and following present in the spool directory?
Could you send the job file?

Unfortunately I don't see the mistake immediately. There is nothing
wrong on your approach.

Alternatively, use the following:
a) ulimit -c unlimited
b) make sure you compile the backend with -g  and -O0 options.

c) execute MyCore --spool /home/<myuser>/Damaris/bin/spool/

d) you will find a core file with the process number

e) gdb MyCore core.nnnnnn

f) type bt to get the stack.
> I'm currently using a lot of printf to find out where it stops working,
> but being able to debug the backend would save a lot of time.
>   
;-), true.

The other way is:
a) gbd MyCore
b) and on the gdb prompt type:  run --spool myspool
